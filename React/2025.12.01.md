```javascript
console.log(returnTrue() && returnFalse());
// &&는 양쪽 다 참이어야 함 아직 오른쪽이 뭔지 모르니까 판단을 보류하고 오른쪽도 확인하러 감

// 처음부터 false가 나오기에 뒤에 true는 확인조차 하지 않아서 
// false가 나옴
```
```javascript
function printName(person) {
    const name =  person && person.name;
    console.log(name || "person의 값이 없음");
}

printName();
printName({ name: "스타벅스" });

// 상황 1: printName(); (아무것도 안 넣음)
person은 undefined입니다.

person && person.name ➡ person이 없으므로 undefined가 name에 저장됨.

name || "대체 문구" ➡ name이 undefined(거짓)이므로 "person의 값이 없음" 출력.

상황 2: printName({ name: "스타벅스" });
person은 객체({...})입니다. (Truthy)

person && person.name ➡ person이 있으므로 뒤에 있는 person.name("스타벅스")이 name에 저장됨.

name || "대체 문구" ➡ name이 "스타벅스"(참)이므로 "스타벅스" 출력.
```
* `[개념 3]` 구조분해할당
```javascript
const func = ({ name, age, hobby, extra }) => {
    console.log(name, age, hobby, extra);
};

func(person);

func(person)을 실행하면서 person이라는 가방(객체)을 던집니다.

함수(func) 입구({...})에서 가방을 통째로 받는 게 아니라, 가방을 열어서 내용물만 쏙쏙 뺍니다.

"가방은 필요 없고, name, age, hobby만 가지고 들어오세요!" 라고 하는 거죠.

그래서 함수 안에서는 person.name 처럼 귀찮게 안 쓰고, 바로 name이라고 쓸 수 있는 겁니다.
```
* `[개념 4]` Spread 연산자
```javascript
let obj1 = {
    a: 1,
    b: 2,
};
let obj2 = {
    ... obj1,
    c: 3,
    d: 4,
};

console.log(obj2); // a: 1   b: 2,   c: 3,   d: 4
// 이런식으로 ... 을 이용해서 객체나 배열의 값을 간편하게 설정 가능
```
* `[개념 5]` Rest 매개변수
```javascript
// 첫 번째 친구는 leader, 나머지는 전부 members로 묶기
function party(leader, ...members) {
    console.log("반장: " + leader);
    console.log("나머지 친구들: " + members);
}

party("신", "철수", "영희", "맹구", "훈이");
// 변수 선언 안하고 ... 뒤에 members 자체가 선언임
// "신" - 반장, 그 외는 나머지 members로 들어감
```
주의할 점: Rest 매개변수가 가장 뒤에 와야함
